phenopacket_mapper.pipeline.load_data_using_data_model(path: str | Path, data_model: DataModel, column_names: Dict[str, str], compliance: Literal['soft', 'hard'] = 'soft') → DataSet[source]ÔÉÅ

Loads data from a file using a DataModel definition

List a column for each field of the DataModel in the column_names dictionary. The keys of the dictionary should be {id}_column for each field and the values should be the name of the column in the file.

E.g.: `python data_model = DataModel("Test data model", [DataField(name="Field 1", value_set=ValueSet())]) column_names = {"field_1_column": "column_name_in_file"} load_data_using_data_model("data.csv", data_model, column_names) `

Parameters:

  • path ‚Äì Path to formatted csv or excel file

  • data_model ‚Äì DataModel to use for reading the file

  • column_names ‚Äì A dictionary mapping from the id of each field of the DataField to the name of a column in the file

  • compliance ‚Äì Compliance level to enforce when reading the file. If ‚Äòsoft‚Äô, the file can have extra fields that are not in the DataModel. If ‚Äòhard‚Äô, the file must have all fields in the DataModel.

List of DataModelInstances

class phenopacket_mapper.pipeline.PhenopacketMapper(datamodel: DataModel)[source]ÔÉÅ

Bases: object

Class to map data using a DataModel to Phenopackets

This class is central to the pipeline for mapping data from a DataModel to Phenopackets. A dataset can be mapped from its tabular format to the Phenopacket schema in a few simple steps: 1. Define the DataModel for the dataset, if it does not exist yet 2. Load the data from the dataset 3. Define the mapping from the DataModel to the Phenopacket schema 4. Perform the mapping 5. Write the Phenopackets to a file 6. Optionally validate the Phenopackets
